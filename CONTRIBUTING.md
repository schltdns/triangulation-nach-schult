Contributing Guidelines
Thank you for your interest in contributing to this project.

I maintain this framework as an individual. Contributions are welcome as long as they respect the project’s integrity, clarity, and epistemic orientation.

Ways to Contribute
I accept contributions in the following forms:

Bug Reports (Issues)  
If you find errors, ambiguities, or inconsistencies, please open an issue with a clear description.

Suggestions for Improvement  
Proposals regarding structure, documentation, or methodological clarity are appreciated.

Pull Requests  
If you propose changes, please submit a pull request with a short explanation and clear commit messages.

Contribution Principles
Please keep in mind:

Respectful and constructive communication

Precise and traceable formulations

No speculative extensions without methodological grounding

No changes that distort the epistemic structure (P1–P8)

No commercial extensions (License: Non‑Commercial)

Pull Request Process
Fork the repository

Create a new branch

Implement your changes

Clearly describe what you changed and why

Submit the pull request

I review each contribution carefully and will respond as soon as possible.

Contact
For questions or suggestions, please open an issue.
