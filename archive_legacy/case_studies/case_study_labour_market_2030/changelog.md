# Changelog – Divergence based Navigation System (DNS)

All notable changes to the TNS methodology, protocol, and case studies.

Format based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).  
Versioning follows semantic versioning: `MAJOR.MINOR‑Open` (Open = public release).

## [1.2-Open] – 2026-04-06

### Added
- **Full case study: Labour Market 2030** (`case_study_labour_market_2030/`)
  - Problem definition, raw model outputs (6 models), divergence analysis
  - Synthesis with Master Frame 3.3, validation protocol, power layer, operator reflection
- **Falsification matrix** for all A–G dimensions (testable criteria with thresholds)
- **Power layer** as fourth, meta‑filter (access, control, profit, risk)
- **Operator reflection** (P7) with bias self‑assessment and falsification criteria for operator decisions
- **Teaching folder** (`teaching/`) with 90‑minute workshop module for IHK/vocational schools

### Changed
- Updated `README.md` with golden sentence and clearer navigation
- Revised protocol documentation to include power layer and falsification rules

### Fixed
- Removed ambiguous “consensus‑first” language – divergence is now explicitly the primary signal
